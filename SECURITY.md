# Security Policy

PooleOS is pre-production research software. It is not currently suitable for protecting production data, operating safety-critical systems, or replacing a supported daily-driver operating system.

## Reporting

Use GitHub private vulnerability reporting for this repository when available. Do not place exploit details, private keys, personal data, unpublished Poole Math methods, or weaponized proofs of concept in a public issue.

If private reporting is unavailable, open a public issue containing only a request for a private contact channel. Rooke Poole is the final severity, disclosure, remediation, and release authority.

## Supported Versions

No version is currently production-supported. Development snapshots, lab images, fixtures, and release candidates receive best-effort review and may change incompatibly.

## Release Claims

Passing a test, schema, static model, simulator, QEMU boot, physical boot, or consistency gate does not establish production security. Only the exact signed ISO and receipt identified by the Production Goal Charter can set `production_ready=true`.
